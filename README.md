# DBMS Project

## Project Overview

This project was developed as part of a Database Management Systems
course. The primary objective of the project is to design, model, and
implement a robust relational database system using sound database
design principles.

The focus of this project is not basic CRUD functionality, but rather:

-   Proper schema design
-   Functional dependency analysis
-   Normalization (1NF to 4NF)
-   Candidate key identification
-   Minimal canonical cover derivation
-   Referential integrity enforcement
-   Query optimization awareness

------------------------------------------------------------------------

## Problem Statement

The system models a real-world scenario with structured relational data.
It ensures data consistency, eliminates redundancy, and enforces
constraints using primary keys, foreign keys, and normalization
techniques.

------------------------------------------------------------------------

## Database Design

### 1. Schema Design

-   Clearly defined relations
-   Proper attribute decomposition
-   Removal of redundancy
-   Logical separation of concerns

### 2. Keys

-   Primary Keys
-   Foreign Keys
-   Candidate Keys (derived using functional dependency analysis)

### 3. Functional Dependencies

-   All relevant functional dependencies identified
-   Armstrong's axioms applied
-   Minimal Canonical Cover derived

### 4. Normal Forms

All relations are validated to satisfy: - 1NF - 2NF - 3NF - BCNF - 4NF
(where applicable)

Justifications are documented in the project report.
